Техническое задание
Разработка AI-агента контроля технического состояния и обслуживания автопарка
1. Общая информация о проекте
AI-агент предназначен для автоматического контроля сроков технического обслуживания, страховки, техосмотра, ремонта, замены расходников и других обязательных мероприятий по каждому автомобилю. Агент анализирует данные по транспортным средствам, пробегу, регламентам обслуживания, срокам документов и истории ремонтов, формирует уведомления, рекомендации и управленческие выводы.
2. Цель проекта
Создание интеллектуального инструмента для контроля эксплуатации автопарка, который снижает риск простоев, штрафов, аварийных ситуаций и незапланированных расходов на срочный ремонт.
3. Проблематика текущего процесса
Контроль сроков обслуживания часто выполняется вручную. Данные по страховке, техосмотру, ремонтам и расходникам могут находиться в разных системах. Отсутствие единого контроля приводит к риску пропуска обязательных мероприятий, увеличению простоев и росту затрат.
4. Целевой процесс
Агент получает данные по автомобилям, анализирует сроки обязательных мероприятий, выявляет риски, отправляет уведомления ответственным сотрудникам и формирует отчеты для руководства.
5. Пользователи системы
Руководитель транспортного подразделения — получает аналитику и риски. Специалист по эксплуатации — контролирует ТО, документы и ремонты. Механик — получает задачи на обслуживание. Водитель — передает информацию о пробеге и неисправностях. Администратор — управляет настройками и доступами.
6. Источники данных
Источники данных: система учета автопарка, Excel/CSV, базы данных, GPS/ГЛОНАСС, система заявок на ремонт, документы по страховке и техосмотру, API внутренних систем.
7. Входные данные
Идентификатор автомобиля, марка, модель, VIN, госномер, год выпуска, подразделение, ответственный, водитель, пробег, данные ТО, страховки, техосмотра, ремонтов, расходников, затраты и комментарии.
8. Основные функции AI-агента
Агент выполняет: контроль технического обслуживания, контроль сроков страховки и техосмотра, анализ ремонтов, контроль замены расходников, анализ состояния автопарка, выявление рисков, прогнозирование будущих мероприятий и формирование рекомендаций.
9. Контроль обслуживания и документов
AI-агент рассчитывает приближение сроков ТО, замены расходников, окончания страховки и техосмотра. При необходимости формирует предупреждения с указанием автомобиля, срока и ответственного сотрудника.
10. Анализ рисков и затрат
Агент выявляет автомобили с высоким риском простоя, просроченными мероприятиями, повторяющимися неисправностями и повышенными затратами на ремонт. Формирует рекомендации по снижению расходов.
11. Уведомления
Уведомления формируются заранее: за 30, 14, 7 и 1 день до события, а также при возникновении просрочки. Каналы: email, корпоративный мессенджер, Telegram-бот или внутренний портал.
12. Формирование управленческих выводов
Агент формирует краткие отчеты: состояние автопарка, ключевые риски, автомобили требующие внимания, предстоящие работы, прогноз расходов и рекомендации для руководства.
13. Архитектура решения
Источники данных → слой интеграции → база данных → модуль аналитики → AI-агент → уведомления, Dashboard и чат-интерфейс.
14. Используемые технологии
Backend: Python, FastAPI, SQLAlchemy, PostgreSQL, Redis, Celery/FastStream. AI: LangChain/LangGraph, LLM, RAG, embeddings, Qdrant/Chroma/FAISS. Frontend: HTML/CSS/JavaScript или React/Vue.
15. Логика AI-агента
Модули агента: сбор данных, нормализация, контроль сроков, анализ ремонтов, прогнозирование рисков, генерация уведомлений и создание управленческих выводов.
16. API
POST /vehicles/import — загрузка автомобилей. 
GET /vehicles — список автомобилей. 
GET /maintenance/upcoming — ближайшие мероприятия. 
GET /maintenance/overdue — просрочки. 
GET /repairs/statistics — статистика ремонтов. 
POST /agent/ask — запрос к AI-агенту. POST /reports/generate — создание отчета.
17. Dashboard
Интерфейс отображает количество автомобилей, состояние автопарка, ближайшие ТО, страховки, техосмотры, ремонты, риски простоя, затраты и выводы AI-агента.
18. MVP
Минимальная версия включает загрузку данных по автомобилям, контроль ТО, страховки и техосмотра, учет ремонтов и расходников, уведомления, чат с агентом и отчет для руководителя.
19. Критерии приемки
Система должна загружать данные, хранить карточки автомобилей, рассчитывать сроки мероприятий, выявлять риски, отправлять уведомления, отвечать на вопросы пользователя и формировать отчеты.
